# Effective Closure

A closed system in this project means **effective closure with respect to the selected model boundary**.

- `closed`: modeled as no relevant exchange channels.
- `effectively_closed`: exchanges may exist physically, but are negligible/controlled for the model objective.
- `open`: exchanges materially affect the modeled entropy accounting.
- `unknown`: boundary assumptions are unresolved.

Nested modeling is represented with `composition` relations, where a subsystem domain is linked to a supersystem domain.
